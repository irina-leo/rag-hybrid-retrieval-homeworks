# RAG / Hybrid Retrieval Homeworks

Учебный репозиторий с домашними заданиями и разбором практических подходов к Retrieval-Augmented Generation.

Проект посвящён сравнению разных стратегий поиска в RAG-системах: dense retrieval через FAISS, лексический поиск BM25, гибридный retrieval, а также иерархический поиск coarse-to-fine.

В качестве корпуса используются нормативные и технические документы, где особенно важно находить точные формулировки, сохранять ссылки на источники и проверять качество извлечения информации.

## Цель проекта

Показать, как разные retrieval-стратегии влияют на качество поиска в RAG:

* как работает dense-поиск по эмбеддингам;
* в каких случаях BM25 лучше находит точные нормативные формулировки;
* как объединять dense и BM25 в hybrid retrieval;
* как строить двухэтапный поиск coarse-to-fine;
* как анализировать ошибки retrieval и улучшать качество выдачи;
* как готовить контекст для ответа LLM с опорой на найденные фрагменты.

## Что реализовано

### 1. Подготовка базы знаний

Реализована обработка текстового корпуса:

* загрузка документов;
* очистка текста;
* разбиение на чанки;
* добавление метаданных: `section`, `source`, `chunk_id`;
* организация документов по разделам;
* подготовка данных для индексации.

### 2. Dense retrieval на FAISS

Собран векторный поиск по чанкам документов:

* генерация эмбеддингов;
* построение FAISS-индекса;
* поиск релевантных фрагментов по смысловой близости;
* сортировка результатов по score;
* подготовка найденных фрагментов для RAG-контекста.

### 3. BM25 retrieval

Добавлен лексический поиск BM25 для точного поиска по ключевым словам и нормативным формулировкам.

BM25 полезен в задачах, где важны:

* точные термины;
* формулировки из документов;
* номера пунктов;
* юридические и технические конструкции;
* совпадение словоформ.

### 4. Hybrid retrieval: FAISS + BM25

Реализован гибридный поиск, который объединяет:

* dense-score из FAISS;
* BM25-score;
* итоговый hybrid-score.

Цель гибридного подхода — совместить смысловой поиск и точное лексическое совпадение, чтобы повысить качество извлечения фрагментов из сложных документов.

### 5. Hierarchical retrieval: coarse-to-fine

Реализован двухэтапный поиск:

1. `coarse` — сначала выбирается релевантный раздел документа;
2. `fine` — затем внутри выбранного раздела ищутся наиболее подходящие чанки.

Такой подход помогает уменьшить шум в выдаче и работать с многоуровневыми документами.

### 6. LlamaIndex multi-index chaining

Дополнительно разобран вариант реализации hierarchical retrieval через LlamaIndex:

* отдельный индекс по разделам;
* отдельные индексы по чанкам внутри каждого раздела;
* многошаговый retrieval;
* ручная сборка контекста для ответа модели.

### 7. Диагностика качества retrieval

В ноутбуках добавлены инструменты отладки:

* таблица top-кандидатов;
* сравнение dense-score, BM25-score и hybrid-score;
* просмотр соседних чанков;
* поиск ожидаемой фразы в корпусе;
* рекомендации по улучшению чанкинга и параметров поиска.

## Примеры исследуемых вопросов

Пример вопроса для проверки retrieval:

> При каких условиях допускается экспертиза доказательной документации вместо испытаний?

Такие вопросы позволяют проверить, способен ли retriever найти точный нормативный фрагмент, а не просто близкий по смыслу текст.
